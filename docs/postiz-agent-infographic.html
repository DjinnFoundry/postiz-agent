Qué problema resuelve

De archivo local a publicación omnicanal.

Postiz publica piezas ya preparadas. Tu pipeline produce audio y metadatos. Este repo es la capa que transforma, valida, adapta por plataforma y deja trazabilidad para runs autónomos.

Convierte audio en video social

Lee MP3, JSON y cover, genera subtítulos karaoke con Whisper y renderiza slides animados con HyperFrames.

Aísla las rarezas de cada plataforma

X, TikTok e Instagram usan Postiz. YouTube delega en YouTubeCLI. Spotify se alimenta con RSS porque no hay publish API por episodio.

Opera sin supervisar sin perder control

`dispatch` elige el siguiente item pendiente, evita duplicados, reintenta fallos transitorios y registra cada decisión en JSONL.

Contrato de entrada

Lo único que debe entregar cualquier proyecto.

postiz-agent no necesita saber si el audio viene de cuentos, podcasts, demos, clases, newsletters o resúmenes internos. Necesita un slug y tres ficheros junto a un JSON mínimo.

Paquete por slug

`slug.mp3` para narración, `slug.json` para metadatos y `slug-cover.png` o una portada fallback por mood.

JSON canónico

`title`, `content`, `mood`, `meta.slug`, `meta.brand`, `meta.generatedAt`, `beats[]` opcional y tags opcionales.

Adapters compatibles

Los aliases antiguos `titulo`, `contenido` y `vocabularioNuevo` siguen entrando, pero se normalizan al contrato genérico.

Flujo principal

Pipeline de `publish` y `render`.

El orquestador transcribe una sola vez y después corre cada plataforma en paralelo. El modo `render` usa el mismo flujo con `dryRun`, así que previsualizar no es un camino alternativo.

Features

Lo que ya trae cableado.

Las features no son adornos: casi todas reducen riesgo operativo de publicar contenido multimedia en varias plataformas desde un proceso programado.

`dispatch` cron-safe

Escanea paquetes completos, consulta historial de 30 días y elige el más antiguo pendiente.

Render previo sin publicar

`render` usa el pipeline real con `dryRun`, ideal para revisar subtítulos y estilo antes del feed.

Subtítulos karaoke

Whisper genera timestamps por palabra; HyperFrames los convierte en slides con highlight sincronizado.

Variantes por plataforma

X 1:1, TikTok e Instagram 9:16, YouTube 16:9. Spotify queda como canal RSS.

Instagram multi-parte

Audios largos se dividen en ventanas de hasta 170s, alineadas a beats o palabras y separadas 5 min.

Guardia anti-duplicados

Si una combinación slug + plataforma ya tuvo éxito en 24h, se salta salvo `--force`.

Retries con backoff

Errores transitorios de red o 5xx se reintentan. Los 4xx no se esconden.

Alertas webhook

Tras agotar retries, dispara una alerta fire-and-forget con slug, plataforma, error e intentos.

Moderación de captions

Blocklist española sobre tokens de Whisper para evitar transcripciones embarazosas.

Decision log JSONL

Memoria append-only para LLMs y humanos: que se intento, por que, resultado y timestamp.

Self-hosted Postiz

Incluye docker-compose y guias OAuth para mantener control de integraciones y scheduling.

CLI agent-readable

`--json` en comandos clave permite encadenarlo con agentes, cron y tooling externo.

Superficies

Un paquete, cinco destinos.

Cada destino tiene un contrato distinto. El código mantiene esas diferencias en publishers pequeños y un registry central.

X 1080 x 1080, 1:1

Video cuadrado vía Postiz. Caption corto con título, marca del JSON y hashtags.

TikTok 1080 x 1920, 9:16

Video vertical vía Postiz, pensado para narración completa y distribución short-form.

Instagram Reels 1080 x 1920, 9:16

Si supera 3 min, divide y agenda partes ordenadas con ribbon `PARTE i/N`.

YouTube 1920 x 1080, 16:9

Renderiza video y delega upload a YouTubeCLI para preservar analytics y log propio.

Spotify / Podcasts RSS + MP3

Genera feed iTunes-compatible. Las plataformas lo consultan por polling.

Autonomia

Como decide que publicar.

`dispatch` no adivina slugs. Solo considera pares `.json + .mp3`, ordena por `meta.generatedAt` o mtime y busca pendientes por plataforma en el decision log.

Fiabilidad

Guardas antes de tocar el feed.

El sistema separa errores recuperables de errores que deben parar el run. La idea de producto: publicar menos, pero no publicar mal.

Arquitectura

Donde vive cada responsabilidad.

El diseño favorece archivos pequeños y responsabilidades claras: orquestación, media, plataformas, logs y utilidades. Agregar una plataforma es un publisher nuevo y una entrada en el registry.

Plantilla visual real

`fantasia` existe. Otros moods caen a `fantasia` con warning en el resultado.

No busca clips virales

Renderiza la pieza completa por defecto. La selección automática de clips queda fuera del alcance actual.

No sube RSS a storage

`rss` genera `feed.xml`; subir feed y MP3 a R2 queda como paso operacional externo.

Traducción producto: postiz-agent no intenta ser un editor creativo generalista. Es una cinta transportadora confiable para convertir audio ya generado en piezas sociales publicables, con suficientes guardas para que cron pueda correr sin estar mirando cada API.
